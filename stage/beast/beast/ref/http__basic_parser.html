During parsing, callbacks will be made to the derived class if those members are present (detected through SFINAE). The signatures which can be present in the derived class are: void on_method(boost::string_ref const&, error_code& ec) Called for each piece of the Request-Method void on_uri(boost::string_ref const&, error_code& ec) Called for each piece of the Request-URI void on_reason(boost::string_ref const&, error_code& ec) Called for each piece of the reason-phrase void on_request(error_code& ec) Called after the entire Request-Line has been parsed successfully. void on_response(error_code& ec) Called after the entire Response-Line has been parsed successfully. void on_field(boost::string_ref const&, error_code& ec) Called for each piece of the current header field. void on_value(boost::string_ref const&, error_code& ec) Called for each piece of the current header value. int on_headers(error_code& ec) Called when all the headers have been parsed successfully. The return value from this callback is special, these are the meanings: 0 : The parser should expect a body 1 : The parser should skip the body. For example, this is used when sending a response to a HEAD request. 2 : The parser should skip ths body, this is an upgrade to a different protocol. void on_body(boost::string_ref const&, error_code& ec) Called for each piece of the body. If the headers indicated chunked encoding, the chunk encoding is removed from the buffer before being passed to the callback. void on_complete(error_code& ec) Called when the entire message has been parsed successfully. At this point, basic_parser::complete() returns true, and the parser is ready to parse another message if keep_alive() would return true. The parser uses traits to determine if the callback is possible. If the Derived type omits the callbacks, they are simply skipped with no compilation error. The default behavior of on_body when the derived class does not provide the member, is to specify that the body should not be skipped. If a callback sets an error, parsing stops at the current octet and the error is returned to the caller.
