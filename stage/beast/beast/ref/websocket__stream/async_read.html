websocket::stream::async_read

Start reading a message asynchronously. 

template<
    class Streambuf,
    class ReadHandler>
void-or-deduced
async_read(
    opcode& op,
    Streambuf& streambuf,
    ReadHandler&& handler);

This function is used to asychronously read a message from the websocket. The function call always returns immediately. Upon a successful completion, op is set to either binary or text depending on the message type, and the input area of the streambuf will hold all the message payload bytes (which may be zero in length).

Parameters

op

A value to receive the message type. This object must remain valid until the handler is called.

streambuf

A stream buffer to hold the message data. This object must remain valid until the handler is called.

handler

The handler to be called when the read operation completes. Copies will be made of the handler as required. The function signature of the handler must be: 

void handler(
    error_code const& error     // Result of operation
);

Regardless of whether the asynchronous operation completes immediately or not, the handler will not be invoked from within this function. Invocation of the handler will be performed in a manner equivalent to using boost::asio::io_service::post().
