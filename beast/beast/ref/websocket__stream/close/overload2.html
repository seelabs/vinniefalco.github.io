websocket::stream::close (2 of 2 overloads)

Perform a WebSocket close. 

void
close(
    close_reason const& cr,
    error_code& ec);

This function initiates the WebSocket close procedure. If the close reason specifies a close code other than close_code::none, the close frame is sent with the close code and optional reason string. Otherwise, the close frame is sent with no payload. Callers should not attempt to write WebSocket data after initiating the close. Instead, callers should continue reading until an error occurs. A read returning error::closed indicates a successful connection closure.

Parameters

cr

The reason for the close.

ec

Set to indicate what error occurred, if any.
